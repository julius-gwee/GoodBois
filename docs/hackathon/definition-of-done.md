# Definition of Done

## Feature Done

A feature is done when:

- It is reachable from the demo flow.
- It uses shared data contracts.
- It has seeded demo data.
- It handles empty/error state enough for the demo.
- It works on mobile width.
- It does not make unsafe or unsupported product claims.
- The relevant docs are updated.

## UI Done

UI is done when:

- Text fits on mobile.
- Buttons are tappable.
- Status is not color-only.
- Elderly mode is simpler than caregiver mode.
- Map information is also available outside the map.

## Safety Done

Safety-sensitive features are done when:

- Consent copy is visible.
- End/stop control is visible.
- The feature says it is not emergency monitoring.
- No medical details are collected.
- Demo data does not include real personal data.

## Integration Done

Integration is done when:

- Map/list/detail use the same resource objects.
- Hazard status appears in the relevant UI.
- Export contains the reviewed hazard data.
- Grab handoff uses selected pickup/drop-off data.
- Route safety uses selected route data.

## Demo Done

Demo is done when one person can run the full scenario without developer explanation or hidden setup.
